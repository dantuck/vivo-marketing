How it works

One command.
Four phases.

1

Backup

restic backs up your directories to a local repo

2

Check

repo integrity is verified before any sync

3

Forget

old snapshots are pruned by your retention policy

4

Sync

data is copied to all configured remote backends

Features

Everything restic
was missing.

🔐

Credentials encrypted at rest

SOPS + age keeps your remote keys out of plaintext configs. Secrets are decrypted in-memory only at runtime.

☁️

Multi-remote sync

Push the same backup to S3, Backblaze B2, MinIO, and more in one run. Add as many remotes as you need.

⏭️

Step control

Use --start-step to skip already-completed phases. Resume from forget, or sync-only runs.

🔄

Self-updating

Run vivo update to pull the latest release. Background version checks notify you after each run.

🔗

Task chaining

Tasks can call other tasks and run shell commands post-backup. Circular references are detected and skipped.

🧪

Dry-run mode

Test your full configuration without writing a single byte. Skips remote sync, passes --dry-run to restic.

Configuration

KDL config.
Human-readable.

Configure backups in a clean KDL file. Set your repo, directories, retention policy, and remotes — all in one place.

Secrets are stored separately in a SOPS-encrypted YAML file and injected as environment variables at runtime.

Default config path

~/.config/vivo/backup.kdl
backup.kdl
// default task to run
default-task "backup"

tasks {
  task "backup" {
    backup {
      repo "$HOME/.local/share/restic/main"
      directory "$HOME"
      retention {
        daily 7
        weekly 5
      }
      remote "s3:https://s3.amazonaws.com/my-bucket" {
        credentials "aws"
      }
    }
    command "notify-send 'Backup complete'"
  }
}
Remote Backends

Your data.
Your clouds.

S3-compatible

AWS S3, MinIO, rustfs, or any S3-compatible endpoint. Uses restic copy for fast, verified transfer.

s3:https://s3.amazonaws.com/bucket

Backblaze B2

Cost-effective cold storage via the b2 sync CLI with automatic replacement and deletion of stale files.

b2:my-bucket:restic/main
Quick Start

Up and running
in four commands.

1

Initialize vivo

Checks prerequisites and scaffolds your config and secrets files.

$ vivo init
2

Edit your config

Set your repo path, directories to back up, and remote destinations.

$ vivo config edit
3

Add your secrets

Set your restic password and remote credentials, encrypted with SOPS + age.

$ vivo secrets edit
4

Run a backup

Dry-run first to verify, then run for real.

$ vivo --dry-run
$ vivo
Install

Get vivo.

One-line install

Linux and macOS. Downloads the latest binary, verifies the checksum.

$ curl -sSf https://raw.githubusercontent.com/dantuck/vivo/main/install.sh | sh

From Cargo

Requires Rust installed. Builds from source and installs to your Cargo bin.

$ cargo install vivo

Build from source

Clone the repo from Codeberg or GitHub and build manually.

$ git clone https://codeberg.org/tuck/vivo
$ cargo build --release
